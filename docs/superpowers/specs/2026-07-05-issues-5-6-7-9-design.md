# Issue 5/6/7/9 设计文档

## 目标

一次解决 GitHub issues #5、#6、#7、#9：

- #9：支持逐层启用/关闭 LRD。
- #7：统一 `sigma_noise` 默认值为 `0.0001`。
- #6：YAML 中的 `lambda_*_init` 生效。
- #5：支持从任意 epoch checkpoint 恢复训练。

## 设计

### #9 逐层 LRD 开关

在 `LRDConfig` 中新增 `layer_enabled: dict` 字段，键为层组名（如 `fc1`），值为 `true/false/auto`。

判定优先级（从高到低）：

1. `layer_generators.{name}.lrd_enabled`（仅 LWT 场景）
2. `lrd.layer_enabled.{name}`
3. 全局 `lrd.enabled`

`TargetNet._should_use_lrd(name, total_params)` 按上述优先级返回是否对该层使用 LRD。

LWT 场景下，`train.py` 将 `layer_generators` 中的 `lrd_enabled` 和 `lrd_rank` 合并进全局 `lrd_config`，再构造 `TargetNet`。

### #7 sigma_noise 默认 0.0001

- 所有 `configs/*.yaml` 中的 `sigma_noise` 统一为 `0.0001`。
- `MappingLoss.__init__` 的默认 `sigma_noise` 从 `0.01` 改为 `0.0001`。

### #6 lambda_*_init 生效

- `MappingLoss.__init__` 新增参数：`lambda_st_init`、`lambda_sm_init`、`lambda_al_init`，默认 `0.1`。
- `train.py` 从 YAML 读取这三个值并传入 `MappingLoss`。

### #5 resume 训练

- `train.py` 和 `train_baseline.py` 增加 `--resume` 参数，接收 checkpoint 路径。
- checkpoint 结构扩展，保存：
  - `state_dict`
  - `optimizer_state_dict`
  - `scheduler_state_dict`
  - `epoch`
  - `best_test_acc`
  - `results`
- `SLVTTrainer`、`LWTTrainer`、`train_baseline.py` 增加 resume 入口或逻辑：
  - 恢复网络/生成器 state_dict
  - 恢复 optimizer、scheduler 状态
  - 从断点 epoch 继续训练
  - 恢复 `best_test_acc` 和 `results`
